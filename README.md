# Custom RV32I RISC-V CPU Simulator & Assembler

> **Note:** This README documentation was AI-generated by Antigravity based on the finalized project architecture.

A fully functional, Turing-complete single-cycle RISC-V CPU Simulator and integrated Two-Pass Assembler, written entirely from scratch in modern C++.

## Overview

This project is a deep dive into computer architecture, systems programming, and compiler design. It models a hardware RISC-V processor (RV32I Base Integer Instruction Set) down to the bitwise level. Rather than relying on external compilers or pre-built emulation libraries, this project implements the entire software-to-hardware pipeline:

1. **The Software Toolchain:** A custom Two-Pass C++ Assembler that reads human-readable `.asm` files, tokenizes them, resolves forward branch labels, and packs them into standard 32-bit RISC-V machine code binaries.
2. **The Hardware Simulator:** A hardware-accurate CPU Execution Engine equipped with 32 General-Purpose Registers, a Program Counter, an Arithmetic Logic Unit (ALU), a hardware Decoder, and a simulated System Bus connecting the CPU to Byte-Addressable RAM.
3. **The Testbench:** A fail-fast, fully automated testing suite that boots a fresh instance of the virtual motherboard, flashes the binary, and verifies execution against the complete RV32I ISA test suite.

## Features

* **Strict RV32I Compliance:** Fully supports all R-Type, I-Type, S-Type, B-Type, U-Type, and J-Type instructions defined in the base unprivileged ISA.
* **Component-Based Architecture:** The project separates concerns cleanly. The `Decoder` unpacks bits, the `ALU` performs math, the `CPU` updates registers, and the `Bus` routes memory requests.
* **Two-Pass Assembler:** Parses code like a real compiler. Resolves forward-declared branch labels correctly and handles pseudo-type complexities like `I_SHIFT` (where shift instructions steal `funct7` space).
* **Automated Verification:** Capable of running bulk `.asm` test scripts to ensure every instruction's silicon-level logic is perfectly sound.

## Architecture

*For a detailed breakdown of the codebase, design philosophy, and component interactions, please read the [ARCHITECTURE.md](ARCHITECTURE.md).*

At a high level, the project acts as a simulated motherboard:
- The **Assembler** translates code into a `.hex` file.
- The **Loader** flashes the `.hex` file onto the **Memory**.
- The **Bus** connects the **CPU** to the **Memory**.
- The **CPU** repeatedly calls `step()`, which Fetches, Decodes, Executes, and Writes-back data until an `ECALL` halts the system.

## Getting Started

1. Write your RISC-V Assembly code in a `.asm` file. Ensure you use standard RV32I syntax.
2. If utilizing the automated testbench, end your successful execution path with `ADDI x17, x0, 10` followed by `ECALL` to gracefully halt the CPU.
3. Pass the path of your `.asm` file to the Assembler in `main.cpp`.
4. Compile and run the C++ project in Visual Studio. The simulator will assemble, load, and execute your code, printing the final register states upon halting.
